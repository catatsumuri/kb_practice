## Markdown 構文ガイド

このアプリがサポートする基本的な Markdown 構文のクイックリファレンスです。見出しやテキスト装飾から、リンク、テーブル、コードブロックまでを扱います。

---

## Markdown とは？

Markdown はプレーンテキストを装飾するための軽量マークアップ言語です。`#`、`*`、`-` のようなシンプルな記号で構造を定義し、それを HTML に変換して表示します。レンダリングする前の状態でも、ソーステキストをそのまま読みやすく保つことが目的です。

---

## 見出し

`#` 記号を使って見出しレベルを定義します。`#` の数がレベルを決めます。

```
# 見出し 1
## 見出し 2
### 見出し 3
#### 見出し 4
```

上記は次のようにレンダリングされます:

#### 見出し 3
##### 見出し 4

> 見出し1はページタイトル用に予約されており、通常ドキュメントごとに1度だけ使用します。

## 段落と改行

#### 段落

段落は **空行** で区切ります。空行を挟まない単なる改行では新しい段落にならず、行が連結されます。

```
これは最初の段落です。

これは2番目の段落です。
```

これは最初の段落です。

これは2番目の段落です。

#### 改行

段落を分けずに段落 **内で** 強制的に改行するには、行末に **半角スペース2つ以上** を入れてから Enter を押します。

```
1行目
2行目（同じ段落内の新しい行）
```

1行目
2行目（同じ段落内の新しい行）

#### 太字

```
**この文字は太字です。**
__これも同じように太字になります。__
```

**この文字は太字です。**
__これも同じように太字になります。__

#### イタリック

```
*この文字はイタリックです。*
_これも同じようにイタリックになります。_
```

*この文字はイタリックです。*
_これも同じようにイタリックになります。_

#### 太字とイタリック

```
***この文字は太字かつイタリックです。***
```

***この文字は太字かつイタリックです。***

#### 取り消し線

```
~~この文字には取り消し線が引かれます。~~
```

#### インラインコード

コードをバッククォート1つで囲むと、等幅フォントのインラインコードとして表示されます。変数名やコマンド、短いスニペットを参照する際に便利です。

```
依存関係をインストールするには `npm install` を実行します。
`<div>` 要素はブロックレベルのコンテナです。
環境変数に `DEBUG=true` を設定します。
```

`<div>` 要素はブロックレベルのコンテナです。

## リスト

#### 番号なしリスト

`-`、`*`、`+` のいずれかで箇条書きを作成できます。これらは相互に入れ替え可能です。

```
- りんご
- オレンジ
- バナナ
```

- りんご
- オレンジ
- バナナ

#### ネストしたリスト

半角スペース2つまたは4つでインデントすると、サブ項目を作成できます。

```
- 果物
  - りんご
  - オレンジ
    - ネーブル
    - ブラッド
- 野菜
  - にんじん
  - ほうれん草
```

- 果物
  - りんご
  - オレンジ
    - ネーブル
    - ブラッド
- 野菜
  - にんじん
  - ほうれん草

#### 番号付きリスト

```
1. 最初のステップ
2. 2番目のステップ
3. 3番目のステップ
```

1. 最初のステップ
2. 2番目のステップ
3. 3番目のステップ

> 実際の番号は関係ありません — Markdown が順番に自動採番します。すべての項目を `1.` にしても正しくレンダリングされます。

#### タスクリスト

未チェックの項目には `- [ ]`、チェック済みの項目には `- [x]` を使います。

```
- [x] 初稿を書く
- [x] コード例を追加する
- [ ] 校正する
- [ ] 公開する
```

- [x] 初稿を書く
- [x] コード例を追加する
- [ ] 校正する
- [ ] 公開する

#### 長い行

長い行は折り返されず横スクロールするため、コードブロックがレイアウトを崩すことはありません。

```sql
SELECT users.id, users.name, orders.id AS order_id, orders.total, orders.status FROM users INNER JOIN orders ON orders.user_id = users.id WHERE orders.status IN ('pending', 'processing') ORDER BY orders.created_at DESC;
```

## リンク

#### インラインリンク

基本形は `[表示テキスト](URL)` です。

```
[Markdown Guide を見る](https://www.markdownguide.org)
```

[Markdown Guide を見る](https://www.markdownguide.org)

#### タイトル付きリンク

URL の後に引用符付きのタイトルを追加します。ホバー時にツールチップとして表示されます。

```
[Markdown Guide](https://www.markdownguide.org "最良の Markdown リファレンス")
```

[Markdown Guide](https://www.markdownguide.org "最良の Markdown リファレンス")

#### 素の URL

URL を山括弧で囲むと、独自の表示テキストなしでクリック可能なリンクになります。

```
<https://www.example.com>
<hello@example.com>
```

<https://www.example.com>
<hello@example.com>

#### 参照形式のリンク

URL を別の場所で定義し、ラベルで参照します。長い URL を本文から分離しておきたい場合に便利です。

```
[GitHub][gh] や [MDN][mdn] のドキュメントを参照してください。

[gh]: https://github.com
[mdn]: https://developer.mozilla.org
```

[GitHub][gh] や [MDN][mdn] のドキュメントを参照してください。

[gh]: https://github.com
[mdn]: https://developer.mozilla.org

## 引用

`>` を使って引用を作成します。`>` を複数重ねるとネストした引用になります。

```
> これは引用です。
> 複数行にまたがることもできます。
>
> 空の `>` 行を挟むと、引用内で段落が分かれます。
>
> > これはネストした引用です。
```

> これは引用です。
> 複数行にまたがることもできます。
>
> 空の `>` 行を挟むと、引用内で段落が分かれます。
>
> > これはネストした引用です。

引用の中には他の Markdown 要素も含められます:

```
> **注記:** これは重要な補足です。
> 引用の中でも自由に `コード` や *強調* を使えます。
```

> **注記:** これは重要な補足です。
> 引用の中でも自由に `コード` や *強調* を使えます。

## テーブル

`|` で列を区切り、ヘッダー区切り行には `-` を使います。

```
| 名前       | 役割      | 有効   |
|------------|-----------|--------|
| Alice      | 管理者    | Yes    |
| Bob        | 編集者    | No     |
| Carol      | 閲覧者    | Yes    |
```

| 名前       | 役割      | 有効   |
|------------|-----------|--------|
| Alice      | 管理者    | Yes    |
| Bob        | 編集者    | No     |
| Carol      | 閲覧者    | Yes    |

#### 列の配置

区切り行に `:` を追加すると、列ごとに配置を制御できます。

```
| 左寄せ     | 中央寄せ   | 右寄せ |
|:-----------|:----------:|-------:|
| Apple      | Banana     | Cherry |
| 1          | 2          | 3      |
```

| 左寄せ     | 中央寄せ   | 右寄せ |
|:-----------|:----------:|-------:|
| Apple      | Banana     | Cherry |
| 1          | 2          | 3      |
